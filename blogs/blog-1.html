Blog 1 - Semantic Markup

Markup

In this blog post I am using various html elements to creat meaning. The only thing I am interested in is what the markup does, not what it looks like. This sentence right here is a bad example. I want the word does to have more impact, so I have used emphasis to make it appear italisised. While this is changing what it looks like, I want it to be emphasised regardless of what I do with the css later on. But this is not a normal use case. Things that are of far more concern are headings, paragraphs, articles, asides, sections, and so forth.

Headings

You will see that I have the entire blog post wrapped in an article tag. This is a good way to use the article element. Having the full post as an article keeps it together and shows that it can stand alone.

There is only one heading 1 on the page. It corresponds to the title in the head, which will increase it's search engine visibility. Below it, I can nest various levels of heading 2, 3, 4, 5, and 6. This is how I structure the content on the page.
